197=head1 NAME
198
199Mojolicious - Real-time web framework
200
201=head1 SYNOPSIS
202
203 # Application
204 package MyApp;
205 use Mojo::Base 'Mojolicious';
206
207 # Route
208 sub startup {
209 my $self = shift;
210 $self->routes->get('/hello')->to('foo#hello');
211 }
212
213 # Controller
214 package MyApp::Controller::Foo;
215 use Mojo::Base 'Mojolicious::Controller';
216
217 # Action
218 sub hello {
219 my $self = shift;
220 $self->render(text => 'Hello World!');
221 }
222
223=head1 DESCRIPTION
224
225An amazing real-time web framework built on top of the powerful L<Mojo> web
226development toolkit. With support for RESTful routes, plugins, commands,
227Perl-ish templates, content negotiation, session management, form validation,
228testing framework, static file server, C<CGI>/C<PSGI> detection, first class
229Unicode support and much more for you to discover.
230
231Take a look at our excellent documentation in L<Mojolicious::Guides>!
232
233=head1 HOOKS
234
235L<Mojolicious> will emit the following hooks in the listed order.
236
237=head2 after_build_tx
238
239Emitted right after the transaction is built and before the HTTP request gets
240parsed.
241
242 $app->hook(after_build_tx => sub {
243 my ($tx, $app) = @_;
244 ...
245 });
246
247This is a very powerful hook and should not be used lightly, it makes some
248rather advanced features such as upload progress bars possible. Note that this
249hook will not work for embedded applications, because only the host application
250gets to build transactions. (Passed the transaction and application object)
251
252=head2 around_dispatch
253
254Emitted right after a new request has been received and wraps around the whole
255dispatch process, so you have to manually forward to the next hook if you want
256to continue the chain. Default exception handling with
257L<Mojolicious::Plugin::DefaultHelpers/"reply-E<gt>exception"> is the first hook
258in the chain and a call to L</"dispatch"> the last, yours will be in between.
259
260 $app->hook(around_dispatch => sub {
261 my ($next, $c) = @_;
262 ...
263 $next->();
264 ...
265 });
266
267This is a very powerful hook and should not be used lightly, it allows you to,
268for example, customize application-wide exception handling, consider it the
269sledgehammer in your toolbox. (Passed a callback leading to the next hook and
270the default controller object)
271
272=head2 before_dispatch
273
274Emitted right before the static file server and router start their work.
275
276 $app->hook(before_dispatch => sub {
277 my $c = shift;
278 ...
279 });
280
281Very useful for rewriting incoming requests and other preprocessing tasks.
282(Passed the default controller object)
283
284=head2 after_static
285
286Emitted after a static file response has been generated by the static file
287server.
288
289 $app->hook(after_static => sub {
290 my $c = shift;
291 ...
292 });
293
294Mostly used for post-processing static file responses. (Passed the default
295controller object)
296
297=head2 before_routes
298
299Emitted after the static file server determined if a static file should be
300served and before the router starts its work.
301
302 $app->hook(before_routes => sub {
303 my $c = shift;
304 ...
305 });
306
307Mostly used for custom dispatchers and collecting metrics. (Passed the default
308controller object)
309
310=head2 around_action
311
312Emitted right before an action gets executed and wraps around it, so you have to
313manually forward to the next hook if you want to continue the chain. Default
314action dispatching is the last hook in the chain, yours will run before it.
315
316 $app->hook(around_action => sub {
317 my ($next, $c, $action, $last) = @_;
318 ...
319 return $next->();
320 });
321
322This is a very powerful hook and should not be used lightly, it allows you for
323example to pass additional arguments to actions or handle return values
324differently. (Passed a callback leading to the next hook, the current
325controller object, the action callback and a flag indicating if this action is
326an endpoint)
327
328=head2 before_render
329
330Emitted before content is generated by the renderer. Note that this hook can
331trigger out of order due to its dynamic nature, and with embedded applications
332will only work for the application that is rendering.
333
334 $app->hook(before_render => sub {
335 my ($c, $args) = @_;
336 ...
337 });
338
339Mostly used for pre-processing arguments passed to the renderer. (Passed the
340current controller object and the render arguments)
341
342=head2 after_render
343
344Emitted after content has been generated by the renderer that will be assigned
345to the response. Note that this hook can trigger out of order due to its
346dynamic nature, and with embedded applications will only work for the
347application that is rendering.
348
349 $app->hook(after_render => sub {
350 my ($c, $output, $format) = @_;
351 ...
352 });
353
354Mostly used for post-processing dynamically generated content. (Passed the
355current controller object, a reference to the content and the format)
356
357=head2 after_dispatch
358
359Emitted in reverse order after a response has been generated. Note that this
360hook can trigger out of order due to its dynamic nature, and with embedded
361applications will only work for the application that is generating the response.
362
363 $app->hook(after_dispatch => sub {
364 my $c = shift;
365 ...
366 });
367
368Useful for rewriting outgoing responses and other post-processing tasks.
369(Passed the current controller object)

586 $app->handler(Mojo::Transaction::HTTP->new);

592=head2 helper
593
594 $app->helper(foo => sub {...});
595
596Add or replace a helper that will be available as a method of the controller
597object and the application object, as well as a function in C<ep> templates. For
598a full list of helpers that are available by default see
599L<Mojolicious::Plugin::DefaultHelpers> and L<Mojolicious::Plugin::TagHelpers>.
600
601 # Helper
602 $app->helper(cache => sub { state $cache = {} });
603
604 # Application
605 $app->cache->{foo} = 'bar';
606 my $result = $app->cache->{foo};
607
608 # Controller
609 $c->cache->{foo} = 'bar';
610 my $result = $c->cache->{foo};
611
612 # Template
613 % cache->{foo} = 'bar';
614 %= cache->{foo}

616=head2 hook
617
618 $app->hook(after_dispatch => sub {...});
619
620Extend L<Mojolicious> with hooks, which allow code to be shared with all
621requests indiscriminately, for a full list of available hooks see L</"HOOKS">.
622
623 # Dispatchers will not run if there's already a response code defined
624 $app->hook(before_dispatch => sub {
625 my $c = shift;
626 $c->render(text => 'Skipped static file server and router!')
627 if $c->req->url->path->to_route =~ /do_not_dispatch/;
628 });

630=head2 new
631
632 my $app = Mojolicious->new;
633 my $app = Mojolicious->new(moniker => 'foo_bar');
634 my $app = Mojolicious->new({moniker => 'foo_bar'});
635
636Construct a new L<Mojolicious> application and call L</"startup">. Will
637automatically detect your home directory and set up logging based on your
638current operating mode. Also sets up the renderer, static file server, a
639default set of plugins and an L</"around_dispatch"> hook with the default
640exception handling.

642=head2 plugin

644 $app->plugin('some_thing');

646 $app->plugin('some_thing', {foo => 23});
647 $app->plugin('SomeThing');

649 $app->plugin('SomeThing', {foo => 23});
650 $app->plugin('MyApp::Plugin::SomeThing');
651 $app->plugin('MyApp::Plugin::SomeThing', foo => 23);
652 $app->plugin('MyApp::Plugin::SomeThing', {foo => 23});
653
654Load a plugin, for a full list of example plugins included in the
655L<Mojolicious> distribution see L<Mojolicious::Plugins/"PLUGINS">.

657=head2 start
658
659 $app->start;
660 $app->start(@ARGV);
661
662Start the command line interface for your application. For a full list of
663commands that are available by default see L<Mojolicious::Commands/"COMMANDS">.
664Note that the options C<-h>/C<--help>, C<--home> and C<-m>/C<--mode>, which are
665shared by all commands, will be parsed from C<@ARGV> during compile time.
666
667 # Always start daemon
668 $app->start('daemon', '-l', 'http://*:8080');

671
672 $app->startup;
673
674This is your main hook into the application, it will be called at application
675startup. Meant to be overloaded in a subclass.
676
677 sub startup {
678 my $self = shift;
679 ...

682=head1 AUTOLOAD
683
684In addition to the L</"ATTRIBUTES"> and L</"METHODS"> above you can also call
685helpers on L<Mojolicious> objects. This includes all helpers from
686L<Mojolicious::Plugin::DefaultHelpers> and L<Mojolicious::Plugin::TagHelpers>.
687Note that application helpers are always called with a new default controller
688object, so they can't depend on or change controller state, which includes
689request, response and stash.
690
691 # Call helper
692 say $app->dumper({foo => 'bar'});
693
694 # Longer version
695 say $app->build_controller->helpers->dumper({foo => 'bar'});
