330=head1 NAME
331
332Mojo::Redis2 - Pure-Perl non-blocking I/O Redis driver
333
334=head1 VERSION
335
3360.26
337
338=head1 DESCRIPTION
339
340L<Mojo::Redis2> is a pure-Perl non-blocking I/O L<Redis|http://redis.io>
341driver for the L<Mojolicious> real-time framework.
342
343=over
344
345Redis is an open source, BSD licensed, advanced key-value cache and store.
346It is often referred to as a data structure server since keys can contain
347strings, hashes, lists, sets, sorted sets, bitmaps and hyperloglogs.
348- L<http://redis.io>.
349
350=back
351
352Features:
353
354=over 4
355
356=item * Blocking support
357
358L<Mojo::Redis2> support blocking methods. NOTE: Calling non-blocking and
359blocking methods are supported on the same object, but might create a new
360connection to the server.
361
362=item * Error handling that makes sense
363
364L<Mojo::Redis> was unable to report back errors that was bound to an operation.
365L<Mojo::Redis2> on the other hand always make sure each callback receive an
366error message on error.
367
368=item * One object for different operations
369
370L<Mojo::Redis> had only one connection, so it could not do more than on
371blocking operation on the server side at the time (such as BLPOP,
372SUBSCRIBE, ...). This object creates new connections pr. blocking operation
373which makes it easier to avoid "blocking" bugs.
374
375=back
376
377=head1 SYNOPSIS
378
379=head2 Blocking
380
381 use Mojo::Redis2;
382 my $redis = Mojo::Redis2->new;
383
384 # Will die() on error.
385 $res = $redis->set(foo => "42"); # $res = OK
386 $res = $redis->get("foo"); # $res = 42
387
388=head2 Non-blocking
389
390 Mojo::IOLoop->delay(
391 sub {
392 my ($delay) = @_;
393 $redis->ping($delay->begin)->get("foo", $delay->begin);
394 },
395 sub {
396 my ($delay, $ping_err, $ping, $get_err, $get) = @_;
397 # On error: $ping_err and $get_err is set to a string
398 # On success: $ping = "PONG", $get = "42";
399 },
400 );
401
402=head2 Pub/sub
403
404L<Mojo::Redis2> can L</subscribe> and re-use the same object to C<publish> or
405run other Redis commands, since it can keep track of multiple connections to
406the same Redis server. It will also re-use the same connection when you
407(p)subscribe multiple times.
408
409 $self->on(message => sub {
410 my ($self, $message, $channel) = @_;
411 });
412
413 $self->subscribe(["some:channel"], sub {
414 my ($self, $err) = @_;
415
416 return $self->publish("myapp:errors" => $err) if $err;
417 return $self->incr("subscribed:to:some:channel");
418 });
419
420=head2 Mojolicious app
421
422 use Mojolicious::Lite;
423
424 helper redis => sub { shift->stash->{redis} ||= Mojo::Redis2->new; };
425
426 get '/' => sub {
427 my $c = shift;
428
429 $c->delay(
430 sub {
431 my ($delay) = @_;
432 $c->redis->get('some:message', $delay->begin);
433 },
434 sub {
435 my ($delay, $err, $message) = @_;
436 $c->render(json => { error => $err, message => $message });
437 },
438 );
439 };
440
441 app->start;

444
445C<$err> in this document is a string containing an error message or
446empty string on success.

474 my ($self, $message, $channel) = @_;

476
477Emitted when a C<$message> is received on a C<$channel> after it has been

485
486Emitted when a C<$message> is received on a C<$channel> matching a
