153=head1 NAME

155Mojo::Transaction::HTTP - HTTP transaction

157=head1 SYNOPSIS
158
159 use Mojo::Transaction::HTTP;
160
161 # Client
162 my $tx = Mojo::Transaction::HTTP->new;
163 $tx->req->method('GET');
164 $tx->req->url->parse('http://example.com');
165 $tx->req->headers->accept('application/json');
166 say $tx->res->code;
167 say $tx->res->headers->content_type;
168 say $tx->res->body;
169 say $tx->remote_address;
170
171 # Server
172 my $tx = Mojo::Transaction::HTTP->new;
173 say $tx->req->method;

175 say $tx->req->headers->accept;
176 say $tx->remote_address;
177 $tx->res->code(200);
178 $tx->res->headers->content_type('text/plain');
179 $tx->res->body('Hello World!');
180
181=head1 DESCRIPTION
182
183L<Mojo::Transaction::HTTP> is a container for HTTP transactions, based on
184L<RFC 7230|http://tools.ietf.org/html/rfc7230> and
185L<RFC 7231|http://tools.ietf.org/html/rfc7231>.
186
187=head1 EVENTS
188
189L<Mojo::Transaction::HTTP> inherits all events from L<Mojo::Transaction> and
190can emit the following new ones.
191
192=head2 request
193
194 $tx->on(request => sub {
195 my $tx = shift;
196 ...
197 });
198
199Emitted when a request is ready and needs to be handled.
200
201 $tx->on(request => sub {
202 my $tx = shift;
203 $tx->res->headers->header('X-Bender' => 'Bite my shiny metal ass!');
204 });
205
206=head2 resume
207
208 $tx->on(resume => sub {
209 my $tx = shift;
210 ...
211 });
212
213Emitted when transaction is resumed.
214
215=head2 unexpected
216
217 $tx->on(unexpected => sub {
218 my ($tx, $res) = @_;
219 ...
220 });
221
222Emitted for unexpected C<1xx> responses that will be ignored.
223
224 $tx->on(unexpected => sub {
225 my $tx = shift;
226 $tx->res->on(finish => sub { say 'Follow-up response is finished.' });
227 });
228
229=head1 ATTRIBUTES
230
231L<Mojo::Transaction::HTTP> inherits all attributes from L<Mojo::Transaction>
232and implements the following new ones.
