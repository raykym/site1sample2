261=head1 NAME
262
263Mojo::UserAgent::Transactor - User agent transactor
264
265=head1 SYNOPSIS
266
267 use Mojo::UserAgent::Transactor;
268
269 # GET request with Accept header
270 my $t = Mojo::UserAgent::Transactor->new;
271 say $t->tx(GET => 'http://example.com' => {Accept => '*/*'})->req->to_string;
272
273 # POST request with form-data
274 say $t->tx(POST => 'example.com' => form => {a => 'b'})->req->to_string;
275
276 # PUT request with JSON data
277 say $t->tx(PUT => 'example.com' => json => {a => 'b'})->req->to_string;
278
279=head1 DESCRIPTION
280
281L<Mojo::UserAgent::Transactor> is the transaction building and manipulation
282framework used by L<Mojo::UserAgent>.
283
284=head1 GENERATORS
285
286These content generators are available by default.
287
288=head2 form
289
290 $t->tx(POST => 'http://example.com' => form => {a => 'b'});
291
292Generate query string, C<application/x-www-form-urlencoded> or
293C<multipart/form-data> content.
294
295=head2 json
296
297 $t->tx(PATCH => 'http://example.com' => json => {a => 'b'});
298
299Generate JSON content with L<Mojo::JSON>.
300
301=head1 ATTRIBUTES
302
303L<Mojo::UserAgent::Transactor> implements the following attributes.
304
305=head2 generators
306
307 my $generators = $t->generators;
308 $t = $t->generators({foo => sub {...}});
309
310Registered content generators, by default only C<form> and C<json> are already
311defined.
312
313=head2 name
314
315 my $name = $t->name;
316 $t = $t->name('Mojolicious');
317
318Value for C<User-Agent> request header of generated transactions, defaults to
319C<Mojolicious (Perl)>.
320
321=head1 METHODS
322
323L<Mojo::UserAgent::Transactor> inherits all methods from L<Mojo::Base> and
324implements the following new ones.
325

327
328 $t = $t->add_generator(foo => sub {...});
329
330Register a content generator.
331
332 $t->add_generator(foo => sub {
333 my ($t, $tx, @args) = @_;
334 ...
335 });
336

338

342

347Actual peer for transaction.

353Build L<Mojo::Transaction::HTTP> proxy C<CONNECT> request for transaction if
354possible.
355
356=head2 redirect
357
358 my $tx = $t->redirect(Mojo::Transaction::HTTP->new);
359
360Build L<Mojo::Transaction::HTTP> follow-up request for C<301>, C<302>, C<303>,
361C<307> or C<308> redirect response if possible.
362
363=head2 tx
364
365 my $tx = $t->tx(GET => 'example.com');
366 my $tx = $t->tx(POST => 'http://example.com');
367 my $tx = $t->tx(GET => 'http://example.com' => {Accept => '*/*'});
368 my $tx = $t->tx(PUT => 'http://example.com' => 'Hi!');
369 my $tx = $t->tx(PUT => 'http://example.com' => form => {a => 'b'});
370 my $tx = $t->tx(PUT => 'http://example.com' => json => {a => 'b'});
371 my $tx = $t->tx(POST => 'http://example.com' => {Accept => '*/*'} => 'Hi!');
372 my $tx = $t->tx(
373 PUT => 'http://example.com' => {Accept => '*/*'} => form => {a => 'b'});
374 my $tx = $t->tx(
375 PUT => 'http://example.com' => {Accept => '*/*'} => json => {a => 'b'});
376
377Versatile general purpose L<Mojo::Transaction::HTTP> transaction builder for
378requests, with support for L</"GENERATORS">.
379
380 # Generate and inspect custom GET request with DNT header and content
381 say $t->tx(GET => 'example.com' => {DNT => 1} => 'Bye!')->req->to_string;
382
383 # Stream response content to STDOUT
384 my $tx = $t->tx(GET => 'http://example.com');
385 $tx->res->content->unsubscribe('read')->on(read => sub { say $_[1] });
386
387 # PUT request with content streamed from file
388 my $tx = $t->tx(PUT => 'http://example.com');
389 $tx->req->content->asset(Mojo::Asset::File->new(path => '/foo.txt'));
390
391The C<json> content generator uses L<Mojo::JSON> for encoding and sets the
392content type to C<application/json>.
393
394 # POST request with "application/json" content
395 my $tx = $t->tx(
396 POST => 'http://example.com' => json => {a => 'b', c => [1, 2, 3]});
397
398The C<form> content generator will automatically use query parameters for
399C<GET> and C<HEAD> requests.
400
401 # GET request with query parameters
402 my $tx = $t->tx(GET => 'http://example.com' => form => {a => 'b'});
403
404For all other request methods the C<application/x-www-form-urlencoded> content
405type is used.
406
407 # POST request with "application/x-www-form-urlencoded" content
408 my $tx = $t->tx(
409 POST => 'http://example.com' => form => {a => 'b', c => 'd'});
410
411Parameters may be encoded with the C<charset> option.
412
413 # PUT request with Shift_JIS encoded form values
414 my $tx = $t->tx(
415 PUT => 'example.com' => form => {a => 'b'} => charset => 'Shift_JIS');
416
417An array reference can be used for multiple form values sharing the same name.
418
419 # POST request with form values sharing the same name
420 my $tx = $t->tx(
421 POST => 'http://example.com' => form => {a => ['b', 'c', 'd']});
422
423A hash reference with a C<content> or C<file> value can be used to switch to
424the C<multipart/form-data> content type for file uploads.
425
426 # POST request with "multipart/form-data" content
427 my $tx = $t->tx(
428 POST => 'http://example.com' => form => {mytext => {content => 'lala'}});
429
430 # POST request with multiple files sharing the same name
431 my $tx = $t->tx(POST => 'http://example.com' =>
432 form => {mytext => [{content => 'first'}, {content => 'second'}]});
433
434The C<file> value should contain the path to the file you want to upload or an
435asset object, like L<Mojo::Asset::File> or L<Mojo::Asset::Memory>.
436
437 # POST request with upload streamed from file
438 my $tx = $t->tx(
439 POST => 'http://example.com' => form => {mytext => {file => '/foo.txt'}});
440
441 # POST request with upload streamed from asset
442 my $asset = Mojo::Asset::Memory->new->add_chunk('lalala');
443 my $tx = $t->tx(
444 POST => 'http://example.com' => form => {mytext => {file => $asset}});
445
446A C<filename> value will be generated automatically, but can also be set
447manually if necessary. All remaining values in the hash reference get merged
448into the C<multipart/form-data> content as headers.
449
450 # POST request with form values and customized upload (filename and header)
451 my $tx = $t->tx(POST => 'http://example.com' => form => {
452 a => 'b',
453 c => 'd',
454 mytext => {
455 content => 'lalala',
456 filename => 'foo.txt',
457 'Content-Type' => 'text/plain'
458 }
459 });
460
461The C<multipart/form-data> content type can also be enforced by setting the
462C<Content-Type> header manually.
463
464 # Force "multipart/form-data"
465 my $headers = {'Content-Type' => 'multipart/form-data'};
466 my $tx = $t->tx(POST => 'example.com' => $headers => form => {a => 'b'});
