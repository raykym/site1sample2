233=head1 SYNOPSIS

235 use Mojo::IOLoop;

238 Mojo::IOLoop->server({port => 3000} => sub {
239 my ($loop, $stream) = @_;
240

243

246

276=head1 DESCRIPTION
277
278L<Mojo::IOLoop> is a very minimalistic event loop based on L<Mojo::Reactor>, it
279has been reduced to the absolute minimal feature set required to build solid
280and scalable non-blocking TCP clients and servers.
281
282Depending on operating system, the default per-process and system-wide file
283descriptor limits are often very low and need to be tuned for better
284scalability. The C<LIBEV_FLAGS> environment variable should also be used to
285select the best possible L<EV> backend, which usually defaults to the not very
286scalable C<select>.
287
288 LIBEV_FLAGS=1 # select
289 LIBEV_FLAGS=2 # poll
290 LIBEV_FLAGS=4 # epoll (Linux)
291 LIBEV_FLAGS=8 # kqueue (*BSD, OS X)
292
293The event loop will be resilient to time jumps if a monotonic clock is
294available through L<Time::HiRes>. A TLS certificate and key are also built
295right in, to make writing test servers as easy as possible. Also note that for
296convenience the C<PIPE> signal will be set to C<IGNORE> when L<Mojo::IOLoop> is
297loaded.
298
299For better scalability (epoll, kqueue) and to provide non-blocking name
300resolution, SOCKS5 as well as TLS support, the optional modules L<EV> (4.0+),
301L<Net::DNS::Native> (0.15+), L<IO::Socket::Socks> (0.64+) and
302L<IO::Socket::SSL> (1.94+) will be used automatically if possible. Individual
303features can also be disabled with the C<MOJO_NO_NDN>, C<MOJO_NO_SOCKS> and
304C<MOJO_NO_TLS> environment variables.
305
306See L<Mojolicious::Guides::Cookbook/"REAL-TIME WEB"> for more.
307

327=head2 max_accepts
328

333shutting down gracefully without interrupting existing connections, defaults to
334C<0>. Setting the value to C<0> will allow this event loop to accept new
335connections indefinitely. Note that up to half of this value can be subtracted
336randomly to improve load balancing between multiple server processes.

342
343The maximum number of accepted connections this event loop is allowed to handle
344concurrently, before stopping to accept new incoming connections, defaults to
345C<1000>.
