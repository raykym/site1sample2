417=head1 DESCRIPTION
418
419L<Mojo::UserAgent> is a full featured non-blocking I/O HTTP and WebSocket user
420agent, with IPv6, TLS, SNI, IDNA, HTTP/SOCKS5 proxy, Comet (long polling),
421keep-alive, connection pooling, timeout, cookie, multipart, gzip compression
422and multiple event loop support.
423
424All connections will be reset automatically if a new process has been forked,
425this allows multiple processes to share the same L<Mojo::UserAgent> object
426safely.
427
428For better scalability (epoll, kqueue) and to provide non-blocking name
429resolution, SOCKS5 as well as TLS support, the optional modules L<EV> (4.0+),
430L<Net::DNS::Native> (0.15+), L<IO::Socket::Socks> (0.64+) and
431L<IO::Socket::SSL> (1.94+) will be used automatically if possible. Individual
432features can also be disabled with the C<MOJO_NO_NDN>, C<MOJO_NO_SOCKS> and
433C<MOJO_NO_TLS> environment variables.
434
435See L<Mojolicious::Guides::Cookbook/"USER AGENT"> for more.
436

641=head2 build_tx

852=head2 websocket
853
854 $ua->websocket('ws://example.com' => sub {...});
855 $ua->websocket(
856 'ws://example.com' => {DNT => 1} => ['v1.proto'] => sub {...});
857
858Open a non-blocking WebSocket connection with transparent handshake, takes the
859same arguments as L<Mojo::UserAgent::Transactor/"websocket">. The callback will
860receive either a L<Mojo::Transaction::WebSocket> or L<Mojo::Transaction::HTTP>
861object, depending on if the handshake was successful.
862
863 $ua->websocket('wss://example.com/echo' => ['v1.proto'] => sub {
864 my ($ua, $tx) = @_;
865 say 'WebSocket handshake failed!' and return unless $tx->is_websocket;
866 say 'Subprotocol negotiation failed!' and return unless $tx->protocol;
867 $tx->on(finish => sub {
868 my ($tx, $code, $reason) = @_;
869 say "WebSocket closed with status $code.";
870 });
871 $tx->on(message => sub {
872 my ($tx, $msg) = @_;
873 say "WebSocket message: $msg";
874 $tx->finish;
875 });
876 $tx->send('Hi!');
877 });
878 Mojo::IOLoop->start unless Mojo::IOLoop->is_running;
879
880You can activate C<permessage-deflate> compression by setting the
881C<Sec-WebSocket-Extensions> header, this can result in much better performance,
882but also increases memory usage by up to 300KB per connection.
883
884 $ua->websocket('ws://example.com/foo' => {
885 'Sec-WebSocket-Extensions' => 'permessage-deflate'
886 } => sub {...});
