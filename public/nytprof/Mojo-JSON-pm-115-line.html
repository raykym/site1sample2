282=head1 NAME
283
284Mojo::JSON - Minimalistic JSON
285
286=head1 SYNOPSIS
287
288 use Mojo::JSON qw(decode_json encode_json);
289
290 my $bytes = encode_json {foo => [1, 2], bar => 'hello!', baz => \1};
291 my $hash = decode_json $bytes;
292
293=head1 DESCRIPTION
294
295L<Mojo::JSON> is a minimalistic and possibly the fastest pure-Perl
296implementation of L<RFC 7159|http://tools.ietf.org/html/rfc7159>.
297
298It supports normal Perl data types like scalar, array reference, hash reference
299and will try to call the C<TO_JSON> method on blessed references, or stringify
300them if it doesn't exist. Differentiating between strings and numbers in Perl
301is hard, depending on how it has been used, a scalar can be both at the same
302time. The string value has a higher precedence unless both representations are
303equivalent.
304
305 [1, -2, 3] -> [1, -2, 3]
306 {"foo": "bar"} -> {foo => 'bar'}
307
308Literal names will be translated to and from L<Mojo::JSON> constants or a
309similar native Perl value.
310
311 true -> Mojo::JSON->true
312 false -> Mojo::JSON->false
313 null -> undef
314
315In addition scalar references will be used to generate booleans, based on if
316their values are true or false.
317
318 \1 -> true
319 \0 -> false
320
321The two Unicode whitespace characters C<u2028> and C<u2029> will always be
322escaped to make JSONP easier, and the character C</> to prevent XSS attacks.
323
324 "\x{2028}\x{2029}</script>" -> "\u2028\u2029<\/script>"
325
326=head1 FUNCTIONS
327
328L<Mojo::JSON> implements the following functions, which can be imported
329individually.
330
331=head2 decode_json
332
333 my $value = decode_json $bytes;
