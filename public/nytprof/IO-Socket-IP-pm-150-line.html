59=head1 NAME
60
61C<IO::Socket::IP> - A drop-in replacement for C<IO::Socket::INET> supporting
62both IPv4 and IPv6
63
64=head1 SYNOPSIS
65
66 use IO::Socket::IP;
67
68 my $sock = IO::Socket::IP->new(
69 PeerHost => "www.google.com",
70 PeerPort => "http",
71 Type => SOCK_STREAM,
72 ) or die "Cannot construct socket - $@";
73
74 my $familyname = ( $sock->sockdomain == PF_INET6 ) ? "IPv6" :
75 ( $sock->sockdomain == PF_INET ) ? "IPv4" :
76 "unknown";
77
78 printf "Connected to google via %s\n", $familyname;
79
80=head1 DESCRIPTION
81
82This module provides a protocol-independent way to use IPv4 and IPv6 sockets,
83as a drop-in replacement for L<IO::Socket::INET>. Most constructor arguments
84and methods are provided in a backward-compatible way. For a list of known
85differences, see the C<IO::Socket::INET> INCOMPATIBILITES section below.
86
87It uses the C<getaddrinfo(3)> function to convert hostnames and service names
88or port numbers into sets of possible addresses to connect to or listen on.
89This allows it to work for IPv6 where the system supports it, while still
90falling back to IPv4-only on systems which don't.
91
92=head1 REPLACING C<IO::Socket> DEFAULT BEHAVIOUR
93
94By placing C<-register> in the import list, C<IO::Socket> uses
95C<IO::Socket::IP> rather than C<IO::Socket::INET> as the class that handles
96C<PF_INET>. C<IO::Socket> will also use C<IO::Socket::IP> rather than
97C<IO::Socket::INET6> to handle C<PF_INET6>, provided that the C<AF_INET6>
98constant is available.
99
100Changing C<IO::Socket>'s default behaviour means that calling the
101C<IO::Socket> constructor with either C<PF_INET> or C<PF_INET6> as the
102C<Domain> parameter will yield an C<IO::Socket::IP> object.
103
104 use IO::Socket::IP -register;
105
106 my $sock = IO::Socket->new(
107 Domain => PF_INET6,
108 LocalHost => "::1",
109 Listen => 1,
110 ) or die "Cannot create socket - $@\n";
111
112 print "Created a socket of type " . ref($sock) . "\n";
113
114Note that C<-register> is a global setting that applies to the entire program;
115it cannot be applied only for certain callers, removed, or limited by lexical
116scope.
117
118=cut

165=head2 $sock = IO::Socket::IP->new( %args )
166
167Creates a new C<IO::Socket::IP> object, containing a newly created socket
168handle according to the named arguments passed. The recognised arguments are:
169
170=over 8
171
172=item PeerHost => STRING
173
174=item PeerService => STRING
175
176Hostname and service name for the peer to C<connect()> to. The service name
177may be given as a port number, as a decimal string.
178
179=item PeerAddr => STRING
180
181=item PeerPort => STRING
182
183For symmetry with the accessor methods and compatibility with
184C<IO::Socket::INET>, these are accepted as synonyms for C<PeerHost> and
185C<PeerService> respectively.
186
187=item PeerAddrInfo => ARRAY
188
189Alternate form of specifying the peer to C<connect()> to. This should be an
190array of the form returned by C<Socket::getaddrinfo>.
191
192This parameter takes precedence over the C<Peer*>, C<Family>, C<Type> and
193C<Proto> arguments.
194
195=item LocalHost => STRING
196
197=item LocalService => STRING
198
199Hostname and service name for the local address to C<bind()> to.
200
201=item LocalAddr => STRING
202
203=item LocalPort => STRING
204
205For symmetry with the accessor methods and compatibility with
206C<IO::Socket::INET>, these are accepted as synonyms for C<LocalHost> and
207C<LocalService> respectively.
208
209=item LocalAddrInfo => ARRAY
210
211Alternate form of specifying the local address to C<bind()> to. This should be
212an array of the form returned by C<Socket::getaddrinfo>.
213
214This parameter takes precedence over the C<Local*>, C<Family>, C<Type> and
215C<Proto> arguments.
216
217=item Family => INT
218
219The address family to pass to C<getaddrinfo> (e.g. C<AF_INET>, C<AF_INET6>).
220Normally this will be left undefined, and C<getaddrinfo> will search using any
221address family supported by the system.
222
223=item Type => INT
224
225The socket type to pass to C<getaddrinfo> (e.g. C<SOCK_STREAM>,
226C<SOCK_DGRAM>). Normally defined by the caller; if left undefined
227C<getaddrinfo> may attempt to infer the type from the service name.
228
229=item Proto => STRING or INT
230
231The IP protocol to use for the socket (e.g. C<'tcp'>, C<IPPROTO_TCP>,
232C<'udp'>,C<IPPROTO_UDP>). Normally this will be left undefined, and either
233C<getaddrinfo> or the kernel will choose an appropriate value. May be given
234either in string name or numeric form.
235
236=item GetAddrInfoFlags => INT
237
238More flags to pass to the C<getaddrinfo()> function. These flags will be
239combined with C<AI_ADDRCONFIG>, and if the C<Listen> argument is given,
240C<AI_PASSIVE>. For more information see the documentation about
241C<getaddrinfo()> in the L<Socket> module.
242
243=item Listen => INT
244
245If defined, puts the socket into listening mode where new connections can be
246accepted using the C<accept> method. The value given is used as the
247C<listen(2)> queue size.
248
249=item ReuseAddr => BOOL
250
251If true, set the C<SO_REUSEADDR> sockopt
252
253=item ReusePort => BOOL
254
255If true, set the C<SO_REUSEPORT> sockopt (not all OSes implement this sockopt)
256
257=item Broadcast => BOOL
258
259If true, set the C<SO_BROADCAST> sockopt
260
261=item V6Only => BOOL
262
263If defined, set the C<IPV6_V6ONLY> sockopt when creating C<PF_INET6> sockets
264to the given value. If true, a listening-mode socket will only listen on the
265C<AF_INET6> addresses; if false it will also accept connections from
266C<AF_INET> addresses.
267
268If not defined, the socket option will not be changed, and default value set
269by the operating system will apply. For repeatable behaviour across platforms
270it is recommended this value always be defined for listening-mode sockets.
271
272Note that not all platforms support disabling this option. Some, at least
273OpenBSD and MirBSD, will fail with C<EINVAL> if you attempt to disable it.
274To determine whether it is possible to disable, you may use the class method
275
276 if( IO::Socket::IP->CAN_DISABLE_V6ONLY ) {
277 ...
278 }
279 else {
280 ...
281 }
282
283If your platform does not support disabling this option but you still want to
284listen for both C<AF_INET> and C<AF_INET6> connections you will have to create
285two listening sockets, one bound to each protocol.
286
287=item Timeout
288
289This C<IO::Socket::INET>-style argument is not currently supported. See the
290C<IO::Socket::INET> INCOMPATIBILITES section below.
291
292=item MultiHomed
293
294This C<IO::Socket::INET>-style argument is ignored, except if it is defined
295but false. See the C<IO::Socket::INET> INCOMPATIBILITES section below.
296
297However, the behaviour it enables is always performed by C<IO::Socket::IP>.
298
299=item Blocking => BOOL
300
301If defined but false, the socket will be set to non-blocking mode. Otherwise
302it will default to blocking mode. See the NON-BLOCKING section below for more
303detail.
304
305=back
306
307If neither C<Type> nor C<Proto> hints are provided, a default of
308C<SOCK_STREAM> and C<IPPROTO_TCP> respectively will be set, to maintain
309compatibility with C<IO::Socket::INET>. Other named arguments that are not
310recognised are ignored.
311
312If the constructor fails, it will set C<$@> to an appropriate error message;
313this may be from C<$!> or it may be some other string; not every failure
314necessarily has an associated C<errno> value.
315
316=head2 $sock = IO::Socket::IP->new( $peeraddr )
317
318As a special case, if the constructor is passed a single argument (as
319opposed to an even-sized list of key/value pairs), it is taken to be the value
320of the C<PeerAddr> parameter. This is parsed in the same way, according to the
321behaviour given in the C<PeerHost> AND C<LocalHost> PARSING section below.
322
323=cut

803=head1 NON-BLOCKING
804
805If the constructor is passed a defined but false value for the C<Blocking>
806argument then the socket is put into non-blocking mode. When in non-blocking
807mode, the socket will not be set up by the time the constructor returns,
808because the underlying C<connect(2)> syscall would otherwise have to block.
809
810The non-blocking behaviour is an extension of the C<IO::Socket::INET> API,
811unique to C<IO::Socket::IP>, because the former does not support multi-homed
812non-blocking connect.
813
814When using non-blocking mode, the caller must repeatedly check for
815writeability on the filehandle (for instance using C<select> or C<IO::Poll>).
816Each time the filehandle is ready to write, the C<connect> method must be
817called, with no arguments. Note that some operating systems, most notably
818C<MSWin32> do not report a C<connect()> failure using write-ready; so you must
819also C<select()> for exceptional status.
820
821While C<connect> returns false, the value of C<$!> indicates whether it should
822be tried again (by being set to the value C<EINPROGRESS>, or C<EWOULDBLOCK> on
823MSWin32), or whether a permanent error has occurred (e.g. C<ECONNREFUSED>).
824
825Once the socket has been connected to the peer, C<connect> will return true
826and the socket will now be ready to use.
827
828Note that calls to the platform's underlying C<getaddrinfo(3)> function may
829block. If C<IO::Socket::IP> has to perform this lookup, the constructor will
830block even when in non-blocking mode.
831
832To avoid this blocking behaviour, the caller should pass in the result of such
833a lookup using the C<PeerAddrInfo> or C<LocalAddrInfo> arguments. This can be
834achieved by using L<Net::LibAsyncNS>, or the C<getaddrinfo(3)> function can be
835called in a child process.
836
837 use IO::Socket::IP;
838 use Errno qw( EINPROGRESS EWOULDBLOCK );
839
840 my @peeraddrinfo = ... # Caller must obtain the getaddinfo result here
841
842 my $socket = IO::Socket::IP->new(
843 PeerAddrInfo => \@peeraddrinfo,
844 Blocking => 0,
845 ) or die "Cannot construct socket - $@";
846
847 while( !$socket->connect and ( $! == EINPROGRESS || $! == EWOULDBLOCK ) ) {
848 my $wvec = '';
849 vec( $wvec, fileno $socket, 1 ) = 1;
850 my $evec = '';
851 vec( $evec, fileno $socket, 1 ) = 1;
852
853 select( undef, $wvec, $evec, undef ) or die "Cannot select - $!";
854 }
855
856 die "Cannot connect - $!" if $!;
857
858 ...
859
860The example above uses C<select()>, but any similar mechanism should work
861analogously. C<IO::Socket::IP> takes care when creating new socket filehandles
862to preserve the actual file descriptor number, so such techniques as C<poll>
863or C<epoll> should be transparent to its reallocation of a different socket
864underneath, perhaps in order to switch protocol family between C<PF_INET> and
865C<PF_INET6>.
866
867For another example using C<IO::Poll> and C<Net::LibAsyncNS>, see the
868F<examples/nonblocking_libasyncns.pl> file in the module distribution.
869

872=head1 C<PeerHost> AND C<LocalHost> PARSING
873
874To support the C<IO::Socket::INET> API, the host and port information may be
875passed in a single string rather than as two separate arguments.
876
877If either C<LocalHost> or C<PeerHost> (or their C<...Addr> synonyms) have any
878of the following special forms, and C<LocalService> or C<PeerService> (or
879their C<...Port> synonyms) are absent, special parsing is applied.
880
881The value of the C<...Host> argument will be split to give both the hostname
882and port (or service name):
883
884 hostname.example.org:http # Host name
885 192.0.2.1:80 # IPv4 address
886 [2001:db8::1]:80 # IPv6 address
887
888In each case, the port or service name (e.g. C<80>) is passed as the
889C<LocalService> or C<PeerService> argument.
890
891Either of C<LocalService> or C<PeerService> (or their C<...Port> synonyms) can
892be either a service name, a decimal number, or a string containing both a
893service name and number, in a form such as
894
895 http(80)
896
897In this case, the name (C<http>) will be tried first, but if the resolver does
898not understand it then the port number (C<80>) will be used instead.
899
900=head2 ( $host, $port ) = IO::Socket::IP->split_addr( $addr )
901
902Utility method that provides the parsing functionality described above.
903Returns a 2-element list, containing either the split hostname and port
904description if it could be parsed, or the given address and C<undef> if it was
905not recognised.
906
907 IO::Socket::IP->split_addr( "hostname:http" )
908 # ( "hostname", "http" )
909
910 IO::Socket::IP->split_addr( "192.0.2.1:80" )
911 # ( "192.0.2.1", "80" )
912
913 IO::Socket::IP->split_addr( "[2001:db8::1]:80" )
914 # ( "2001:db8::1", "80" )
915
916 IO::Socket::IP->split_addr( "something.else" )
917 # ( "something.else", undef )
918
919=cut
920
921sub split_addr
922{
923 shift;
924 my ( $addr ) = @_;
925
926 local ( $1, $2 ); # Placate a taint-related bug; [perl #67962]
927 if( $addr =~ m/\A\[($IPv6_re)\](?::([^\s:]*))?\z/ or
928 $addr =~ m/\A([^\s:]*):([^\s:]*)\z/ ) {
929 return ( $1, $2 ) if defined $2 and length $2;
930 return ( $1, undef );
931 }
932
933 return ( $addr, undef );
934}
