302=head1 NAME
303
304Mojolicious::Plugin::OAuth2 - Auth against OAuth2 APIs
305
306=head1 DESCRIPTION
307
308This Mojolicious plugin allows you to easily authenticate against a
309L<OAuth2|http://oauth.net> provider. It includes configurations for a few
310popular providers, but you can add your own easily as well.
311
312Note that OAuth2 requires https, so you need to have the optional Mojolicious
313dependency required to support it. Run the command below to check if
314L<IO::Socket::SSL> is installed.
315
316 $ mojo version
317
318=head2 Breaking changes
319
320Between 1.51 (2015-03-18) and 1.53 (2015-09-08) we decided to remove the back
321compat hack for L</get_token>, making it act I<only> as documented.
322
323The two helpers C<get_authorize_url()> and C<get_token()> will be removed
324soon. They have been deprecated since 1.4 (2015-03-01).
325
326=head2 References
327
328=over 4
329
330=item * L<http://oauth.net/documentation/>

332=item * L<http://aaronparecki.com/articles/2012/07/29/1/oauth2-simplified>
333
334=item * L<http://homakov.blogspot.jp/2013/03/oauth1-oauth2-oauth.html>
335
336=item * L<http://en.wikipedia.org/wiki/OAuth#OAuth_2.0>

338=back

458To enable a "mocked" OAuth2 api, you need to give the special "mocked"

539=head2 oauth2.get_token
540
541 $data = $c->oauth2->get_token($provider_name => \%args);
542 $c = $c->oauth2->get_token($provider_name => \%args, sub {
543 my ($c, $err, $data) = @_;
544 # do stuff with $data->{access_token} if it exists.
545 });
546
547L</oauth2.get_token> is used to either fetch access token from OAuth2 provider,
548handle errors or redirect to OAuth2 provider. This method can be called in either
549blocking or non-blocking mode. C<$err> holds a error description if something
550went wrong. Blocking mode will C<die($err)> instead of returning it to caller.
551C<$data> is a hash-ref containing the access token from the OAauth2 provider.
552C<$data> in blocking mode can also be C<undef> if a redirect has been issued
553by this module.
554
555In more detail, this method will do one of two things:
556
557=over 4
558
559=item 1.
560
561If called from an action on your site, it will redirect you to the
562C<$provider_name>'s C<authorize_url>. This site will probably have some
563sort of "Connect" and "Reject" button, allowing the visitor to either
564connect your site with his/her profile on the OAuth2 provider's page or not.
565
566=item 2.
567
568The OAuth2 provider will redirect the user back to your site after clicking the
569"Connect" or "Reject" button. C<$data> will then contain a key "access_token"
570on "Connect" and a false value (or die in blocking mode) on "Reject".
571
572=back
573
574The method takes these arguments: C<$provider_name> need to match on of
575the provider names under L</Configuration> or a custom provider defined
576when L<registering|/SYNOPSIS> the plugin.
577
578C<%args> can have:
579
580=over 4
581
582=item * host
583
584Useful if your provider uses different hosts for accessing different accounts.
585The default is specified in the provider configuration.
586
587=item * scope
588
589Scope to ask for credentials to. Should be a space separated list.
590
591=back
