286
287=head1 SYNOPSIS
288
289 use Mojo::Template;
290
291 # Use Perl modules
292 my $mt = Mojo::Template->new;
293 say $mt->render(<<'EOF');
294 % use Time::Piece;
295 <div>
296 % my $now = localtime;
297 Time: <%= $now->hms %>
298 </div>
299 EOF
300
301 # Render with arguments
302 say $mt->render(<<'EOF', [1 .. 13], 'Hello World!');
303 % my ($numbers, $title) = @_;
304 <div>
305 <h1><%= $title %></h1>
306 % for my $i (@$numbers) {
307 Test <%= $i %>
308 % }
309 </div>
310 EOF
311
312 # Render with named variables
313 say $mt->vars(1)->render(<<'EOF', {title => 'Hello World!'});
314 <div>
315 <h1><%= $title %></h1>
316 %= 5 + 5
317 </div>
318 EOF

320=head1 DESCRIPTION
321
322L<Mojo::Template> is a minimalistic, fast, and very Perl-ish template engine,
323designed specifically for all those small tasks that come up during big
324projects. Like preprocessing a configuration file, generating text from heredocs
325and stuff like that.
326
327See L<Mojolicious::Guides::Rendering> for information on how to generate
328content with the L<Mojolicious> renderer.
329
330=head1 SYNTAX
331
332For all templates L<strict>, L<warnings>, L<utf8> and Perl 5.10
333L<features|feature> are automatically enabled.
334
335 <% Perl code %>
336 <%= Perl expression, replaced with result %>
337 <%== Perl expression, replaced with XML escaped result %>
338 <%# Comment, useful for debugging %>
339 <%% Replaced with "<%", useful for generating templates %>
340 % Perl code line, treated as "<% line =%>" (explained later)
341 %= Perl expression line, treated as "<%= line %>"
342 %== Perl expression line, treated as "<%== line %>"
343 %# Comment line, useful for debugging
344 %% Replaced with "%", useful for generating templates
345
346Escaping behavior can be reversed with the L</"auto_escape"> attribute, this is
347the default in L<Mojolicious> C<.ep> templates, for example.
348
349 <%= Perl expression, replaced with XML escaped result %>
350 <%== Perl expression, replaced with result %>
351
352L<Mojo::ByteStream> objects are always excluded from automatic escaping.
353
354 % use Mojo::ByteStream 'b';
355 <%= b('<div>excluded!</div>') %>
356
357Whitespace characters around tags can be trimmed by adding an additional equal
358sign to the end of a tag.
359
360 <% for (1 .. 3) { %>
361 <%= 'Trim all whitespace characters around this expression' =%>
362 <% } %>
363
364Newline characters can be escaped with a backslash.
365
366 This is <%= 1 + 1 %> a\
367 single line
368
369And a backslash in front of a newline character can be escaped with another
370backslash.
371
372 This will <%= 1 + 1 %> result\\
373 in multiple\\
374 lines
375
376You can capture whole template blocks for reuse later with the C<begin> and
377C<end> keywords. Just be aware that both keywords are part of the surrounding
378tag and not actual Perl code, so there can only be whitespace after C<begin>
379and before C<end>.
380
381 <% my $block = begin %>
382 <% my $name = shift; =%>
383 Hello <%= $name %>.
384 <% end %>
385 <%= $block->('Baerbel') %>
386 <%= $block->('Wolfgang') %>
387
388Perl lines can also be indented freely.
389
390 % my $block = begin
391 % my $name = shift;
392 Hello <%= $name %>.
393 % end
394 %= $block->('Baerbel')
395 %= $block->('Wolfgang')
396
397L<Mojo::Template> templates get compiled to a Perl subroutine, that means you
398can access arguments simply via C<@_>.
399
400 % my ($foo, $bar) = @_;
401 % my $x = shift;
402 test 123 <%= $foo %>
403
404The compilation of templates to Perl code can make debugging a bit tricky, but
405L<Mojo::Template> will return L<Mojo::Exception> objects that stringify to
406error messages with context.
407
408 Bareword "xx" not allowed while "strict subs" in use at template line 4.
409 2: </head>
410 3: <body>
411 4: % my $i = 2; xx
412 5: %= $i * 2
413 6: </body>
414
415=head1 ATTRIBUTES
416
417L<Mojo::Template> implements the following attributes.
418
419=head2 auto_escape
420
421 my $bool = $mt->auto_escape;
422 $mt = $mt->auto_escape($bool);
423
424Activate automatic escaping.
425
426 # "&lt;html&gt;"
427 Mojo::Template->new(auto_escape => 1)->render("<%= '<html>' %>");
428
429=head2 append
430
431 my $code = $mt->append;
432 $mt = $mt->append('warn "Processed template"');
433
434Append Perl code to compiled template. Note that this code should not contain
435newline characters, or line numbers in error messages might end up being wrong.
436
437=head2 capture_end
438
439 my $end = $mt->capture_end;
440 $mt = $mt->capture_end('end');
441
442Keyword indicating the end of a capture block, defaults to C<end>.
443
444 <% my $block = begin %>
445 Some data!
446 <% end %>
447
448=head2 capture_start
449
450 my $start = $mt->capture_start;
451 $mt = $mt->capture_start('begin');
452
453Keyword indicating the start of a capture block, defaults to C<begin>.
454
455 <% my $block = begin %>
456 Some data!
457 <% end %>
458
459=head2 code
460
461 my $code = $mt->code;
462 $mt = $mt->code($code);
463
464Perl code for template if available.
465
466=head2 comment_mark
467
468 my $mark = $mt->comment_mark;
469 $mt = $mt->comment_mark('#');
470
471Character indicating the start of a comment, defaults to C<#>.
472
473 <%# This is a comment %>
474
475=head2 compiled
476
477 my $compiled = $mt->compiled;
478 $mt = $mt->compiled($compiled);
479
480Compiled template code if available.
481
482=head2 encoding
483
484 my $encoding = $mt->encoding;
485 $mt = $mt->encoding('UTF-8');
486
487Encoding used for template files, defaults to C<UTF-8>.
488
489=head2 escape
490
491 my $cb = $mt->escape;
492 $mt = $mt->escape(sub {...});
493
494A callback used to escape the results of escaped expressions, defaults to
495L<Mojo::Util/"xml_escape">.
496
497 $mt->escape(sub {
498 my $str = shift;
499 return reverse $str;
500 });
501
502=head2 escape_mark
503
504 my $mark = $mt->escape_mark;
505 $mt = $mt->escape_mark('=');
506
507Character indicating the start of an escaped expression, defaults to C<=>.
508
509 <%== $foo %>
510
511=head2 expression_mark
512
513 my $mark = $mt->expression_mark;
514 $mt = $mt->expression_mark('=');
515
516Character indicating the start of an expression, defaults to C<=>.
517
518 <%= $foo %>
519
520=head2 line_start
521
522 my $start = $mt->line_start;
523 $mt = $mt->line_start('%');
524
525Character indicating the start of a code line, defaults to C<%>.
526
527 % $foo = 23;
528
529=head2 name
530
531 my $name = $mt->name;
532 $mt = $mt->name('foo.mt');
533
534Name of template currently being processed, defaults to C<template>. Note that
535this value should not contain quotes or newline characters, or error messages
536might end up being wrong.
537
538=head2 namespace
539
540 my $namespace = $mt->namespace;
541 $mt = $mt->namespace('main');
542
543Namespace used to compile templates, defaults to C<Mojo::Template::SandBox>.
544Note that namespaces should only be shared very carefully between templates,
545since functions and global variables will not be cleared automatically.
546
547=head2 prepend
548
549 my $code = $mt->prepend;
550 $mt = $mt->prepend('my $self = shift;');
551
552Prepend Perl code to compiled template. Note that this code should not contain
553newline characters, or line numbers in error messages might end up being wrong.
554
555=head2 replace_mark
556
557 my $mark = $mt->replace_mark;
558 $mt = $mt->replace_mark('%');
559
560Character used for escaping the start of a tag or line, defaults to C<%>.
561
562 <%% my $foo = 23; %>
563
564=head2 tag_start
565
566 my $start = $mt->tag_start;
567 $mt = $mt->tag_start('<%');
568
569Characters indicating the start of a tag, defaults to C<E<lt>%>.
570
571 <% $foo = 23; %>
572
573=head2 tag_end
574
575 my $end = $mt->tag_end;
576 $mt = $mt->tag_end('%>');
577
578Characters indicating the end of a tag, defaults to C<%E<gt>>.
579
580 <%= $foo %>
581
582=head2 tree
583
584 my $tree = $mt->tree;
585 $mt = $mt->tree([['text', 'foo'], ['line']]);
586
587Template in parsed form if available. Note that this structure should only be
588used very carefully since it is very dynamic.
589
590=head2 trim_mark
591
592 my $mark = $mt->trim_mark;
593 $mt = $mt->trim_mark('-');
594
595Character activating automatic whitespace trimming, defaults to C<=>.
596
597 <%= $foo =%>
598
599=head2 unparsed
600
601 my $unparsed = $mt->unparsed;
602 $mt = $mt->unparsed('<%= 1 + 1 %>');
603
604Raw unparsed template if available.
605
606=head2 vars
607
608 my $bool = $mt->vars;
609 $mt = $mt->vars($bool);
610
611Instead of a list of values, use a hash reference with named variables to pass
612data to templates.
613
614 # "works!"
615 Mojo::Template->new(vars => 1)->render('<%= $test %>!', {test => 'works'});
616
617=head1 METHODS
618
619L<Mojo::Template> inherits all methods from L<Mojo::Base> and implements the
620following new ones.
621
622=head2 parse
623
624 $mt = $mt->parse('<%= 1 + 1 %>');
625
626Parse template into L</"tree">.
627
628=head2 process
629
630 my $output = $mt->process;
631 my $output = $mt->process(@args);
632 my $output = $mt->process({foo => 'bar'});
633
634Process previously parsed template and return the result, or a
635L<Mojo::Exception> object if rendering failed.
636
637 # Parse and process
638 say Mojo::Template->new->parse('Hello <%= $_[0] %>')->process('Bender');
639
640 # Reuse template (for much better performance)
641 my $mt = Mojo::Template->new;
642 say $mt->render('Hello <%= $_[0] %>!', 'Bender');
643 say $mt->process('Fry');
644 say $mt->process('Leela');
645
646=head2 render
647
648 my $output = $mt->render('<%= 1 + 1 %>');
649 my $output = $mt->render('<%= shift() + shift() %>', @args);
650 my $output = $mt->render('<%= $foo %>', {foo => 'bar'});
651
652Render template and return the result, or a L<Mojo::Exception> object if
653rendering failed.
654
655 # Longer version
656 my $output = $mt->parse('<%= 1 + 1 %>')->process;
657
658 # Render with arguments
659 say Mojo::Template->new->render('<%= $_[0] %>', 'bar');
660
661 # Render with named variables
662 say Mojo::Template->new(vars => 1)->render('<%= $foo %>', {foo => 'bar'});
663
664=head2 render_file
665
666 my $output = $mt->render_file('/tmp/foo.mt');
667 my $output = $mt->render_file('/tmp/foo.mt', @args);
668 my $output = $mt->render_file('/tmp/bar.mt', {foo => 'bar'});
669
670Same as L</"render">, but renders a template file.
671
