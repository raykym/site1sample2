92=head1 DESCRIPTION
93
94L<Mojolicious::Plugin::Config> is a Perl-ish configuration plugin.
95
96The application object can be accessed via C<$app> or the C<app> function,
97L<strict>, L<warnings>, L<utf8> and Perl 5.10 L<features|feature> are
98automatically enabled. A default configuration filename in the application home
99directory will be generated from the value of L<Mojolicious/"moniker">
100(C<$moniker.conf>). You can extend the normal configuration file
101C<$moniker.conf> with C<mode> specific ones like C<$moniker.$mode.conf>, which
102will be detected automatically.
103
104The code of this plugin is a good example for learning to build new plugins,
105you're welcome to fork it.
106
107See L<Mojolicious::Plugins/"PLUGINS"> for a list of plugins that are available
108by default.
109
110=head1 OPTIONS
111
112L<Mojolicious::Plugin::Config> supports the following options.
113
114=head2 default
115
116 # Mojolicious::Lite
117 plugin Config => {default => {foo => 'bar'}};
118
119Default configuration, making configuration files optional.
120
121=head2 ext
122
123 # Mojolicious::Lite
124 plugin Config => {ext => 'stuff'};
125
126File extension for generated configuration filenames, defaults to C<conf>.
127
128=head2 file
129
130 # Mojolicious::Lite
131 plugin Config => {file => 'myapp.conf'};
132 plugin Config => {file => '/etc/foo.stuff'};
133
134Path to configuration file, absolute or relative to the application home
135directory, defaults to the value of the C<MOJO_CONFIG> environment variable or
136C<$moniker.conf> in the application home directory.
137
